5.Nested routes
Nested Routes
👉 Used when pages share a common layout (Navbar, Sidebar, Footer)
🧠 Example
/dashboard
/dashboard/profile
/dashboard/settings

Folder structure
pages/
 └─ Dashboard.jsx
 └─ Profile.jsx
 └─ Settings.jsx

 Dashboard.jsx (Parent)
 import { Outlet, Link } from 'react-router-dom'

function Dashboard() {
  return (
    <>
      <h2>Dashboard</h2>
      <Link to="profile">Profile</Link>
      <Link to="settings">Settings</Link>

      <Outlet /> {/* Child routes render here */}
    </>
  )
}

export default Dashboard

import { Routes, Route } from 'react-router-dom'
import Dashboard from './pages/Dashboard'
import Profile from './pages/Profile'
import Settings from './pages/Settings'

function App() {
  return (
    <Routes>
      <Route path="/dashboard" element={<Dashboard />}>
        <Route path="profile" element={<Profile />} />
        <Route path="settings" element={<Settings />} />
      </Route>
    </Routes>
  )
}

export default App

📌 Outlet = placeholder for child components.


6.Protected routes (auth)
👉 Used to block pages unless user is logged in
ProtectedRoute.jsx
import { Navigate } from 'react-router-dom'

function ProtectedRoute({ children }) {
  const isLoggedIn = false // replace with auth logic

  return isLoggedIn ? children : <Navigate to="/login" />
}

export default ProtectedRoute

Usage in App.jsx
<Route
  path="/dashboard"
  element={
    <ProtectedRoute>
      <Dashboard />
    </ProtectedRoute>
  }
/>

8.Layout routes
👉 Used when multiple pages share same UI
(navbar, footer, sidebar)
Layout.jsximport { Outlet } from 'react-router-dom'
import Navbar from '../components/Navbar'

function Layout() {
  return (
    <>
      <Navbar />
      <Outlet />
    </>
  )
}

export default Layout

App.jsx<Route element={<Layout />}>
  <Route path="/" element={<Home />} />
  <Route path="/about" element={<About />} />
</Route>

9.Difference between <a> tag and <Link> in React Router

<a href="/about">About</a>
What happens?
Browser makes a full page reload
React app restarts
State is lost
Slower navigation

When to use <a>?
✅ External links
<a href="https://google.com" target="_blank">Google</a>

import { Link } from 'react-router-dom'
<Link to="/about">About</Link>

What happens?
No page reload
React Router updates URL
Only component changes
App state is preserved
Faster (SPA behavior)
<Link> prevents page reload and enables client-side routing, while <a> causes a full page refresh.
